Edge Commons Core Features

Overview of the basic Dirty little helpers

This section will give you a summary of the currently available features and helper functions of the Edge Commons (2.0.0) release (within the complete All in One package). Remember that you may choose to use only parts of it with the different modules. Consequently, this section is divided in a similar way, starting with the most important Core features here and then moving on to the different modules, i.e. Parallax, Spotlight and SVG throughout the next pages.

The subsequent features are all part of the Edge Commons core and therefore always available as soon as you integrate the Edge Commons into your project. Most of these features are useful helper functions that might help you in any given project.

Add custom CSS rules at runtime

EC.addCSS();
This function doesn't add CSS styles inline, but defines global CSS rules like you would have with an externally linked CSS file. You can apply the CSS rules to elements in Animate by added CSS classes in the property inspector. Use the !important statement on property level to prevent Animate to override the property.

Example: Adding a CSS class at runtime

In this example we define the style for a class "custom-box" that we can assign to any element. The style will then be applied at runtime. The first parameter is the selector (here: .custom-box), followed by the CSS properties as an object in curly braces. 

          
// Add CSS rule for class 'custom-box'
EC.addCSS(".custom-box", {
 "background-color": "rgba(217,68,148,1.00) !important",
 "border-radius": "16px"
});

Click on the button in the demo project below to see how the CSS styles can be applied to the different boxes, for instance both elements on the left and right side ("custom-box"). It is the CSS that we defined in the code above. Note that the click is only for demontration purposes. Usually the styles would be applied as soon as the elements get loaded.

Center an element

The purpose of this helper function is to center an element and keep it centered. This function even works with a flexible stage (percentage-based). Known issues:
+ If the affected element resizes you have to call this function again
+ Does not consider rotation/scaling
+ Does currently NOT work with a symbol element (since we are not considering transform/translate).

Example: Centering an element

This helper function allows you to center a element (in regard to its parent). See the demo projects below to see it in action. 

          
// Center element named "myElement"
EC.centerElement( sym.$("myElement") );

Advanced Logging

Unfortunately, Edge Animate doesn't offer much in regards to finding errors in your code. Especially with more complex projects, it is a good thing to have a variety of advanced logging options at hand. The Edge Commons offer you four different notification types: debug, error, info and warn. They allow you to send a message to the console or a custom target for logging and debugging, each targeting the equivalent mode.

Example: Logging in debug mode

Here is an example of how to send a message in debug mode. The first parameter is the logging message that you want to give out, the second the logging group. In this case this group is stored in the variable LOG_GROUP. 

          
// Log in debug mode
var LOG_GROUP = "My Module";
EC.debug( "Hello logger", LOG_GROUP );

Get reference to all children of a symbol

The function EC.getChildSymbols(); returns all children of a symbol. You can decide between recursive to return all children or not recursive (= false) to return immediate children only.

Example: Logging all children of the stage and a symbol

In this scenario we want to get information about the structure of a composition. Therefore we use console.log(); to display the return value of our helperfunction in the console. We use three different variations to a) get all the children of the stage, b) only the immediate children of the stage and c) get all the children of a symbol on the stage. 

          
// Usage
console.log( "stage: ", getChildSymbols(sym) );
console.log( "stage: ", getChildSymbols(sym, false) );
console.log( "MySymbol1: ", getChildSymbols(sym.getSymbol("MySymbol1")) );

Get injected data

EC.getInjectedData();
Use EC.getInjectedData(); to read injected data (JSON formatted) from the HTML wrapper of the stage. This is a great helper function for dynamic content and CMS integration.
Attention: This feature requires jQuery to be loaded. Get the latest jQuery library here: http://code.jquery.com.

Example: Getting injected data (title and message)

In this case we want to grab the information stored in a data object in the HTML file and render it in our Animate composition, in the corresponding text elements. 

          
// The HTML of the composition with data injection looks like this:
// <div id="Stage" class="EDGE-145692202">
//   <script class="data" type="text/text">
//   { 
//     "title": "I'm injected", 
//     "message": "This text was injected from HTML"
//   }
//   </script>
// </div>   

// Read injected data from HTML into Animate composition. 
var data = EC.getInjectedData(sym);

// Render injected data
sym.$("txtTitle").html( data.title );
sym.$("txtMessage").html( data.message );

Get the name of a symbol

EC.getSymbolName();
Get the full name of a symbol (e.g. Stage_ResponsiveElement). This function is handy if you are working with dynamically created symbols and you don't define the symbol's name yourself. (If name should be used in sym.getSymbol(NAME) the preceding "#" is necessary)

Example: Getting the name of a dynamically created symbol

In this scenario we create a new symbol dynamically with Edge Animate's sym.createChildSymbol(); function. We then need our Edge Commons helper function to get a reference to this newly created symbol instance in order to e.g. play its timeline: 

          
// Create symbol dynamically
var newSymbol = sym.createChildSymbol( "MyButton", sym.$("navigation") );
// Get the name of a symbol
var name = EC.getSymbolName( sym.getSymbol(newSymbol) );
// Use symbol name to get reference to a symbol instance 
//(e.g. to control the timeline)
sym.getSymbol( name ).play();

Find out if full jQuery is used

With EC.isJQuery(); you can find out if full jQuery is available. In that case, the return value of this function is true.

Example: Logging whether full jQuery is in use

Here we use a console log and the Edge Commons helper function to determine whether a full version of jQuery is available: 

          
// Check if full jQuery is in use
console.log( "jQuery is available: ", EC.isJQuery() );

Detect a mobile environment

EC.isMobile();
If you want to find out if the composition is running on a mobile device, you can use this function (e.g. to switch to gestures instead of mouse input).
Mobile user agents supported: Android, iPhone/iPad/iPod, Opera Mini and IEMobile.
Also consider using EC.CLICK_OR_TOUCH constant for event handler.

Example: Checking if composition is running on a mobile device

We use a console log and the Edge Commons helper function to determine whether the composition is running on a mobile device. The return value is true when the device is mobile, otherwise false. 

          
// Is composition running on a mobile device?
console.log( "This is a mobile device: ", EC.isMobile() );

Load external compositions

EC.loadComposition();
This helper allows you to load an Animate composition into a symbol or element.
As parameters you have to hand in the path to the HTML file of the external composition as string and the target container (element or symbol). Optional: You can also add a callback function that is called when loading the composition is completed and the external composition is fully built.

Example: Loading an external composition into a symbol

In this example we load the external composition "sub2.html" into the symbol "targetContainer". Therefore we first store a reference to that target symbol in a variable and hand in the path and the reference to the helper function. After the composition is loaded we can use the callback to change the inner HTML and append text. 

          
var targetContainer = sym.getSymbol("targetContainer");
EC.loadComposition("sub2.html", targetContainer)
  .done( function(comp) {
     comp.getStage().$("mytext").html("hello number 2");
     comp.getStage().$('targetContainer').append("Hello Composition");
  });

Create animated buttons

EC.makeAnimatedButton();
Convert any given symbol into an animated button component that will be played forwards and backwards on mouse events. An optional "hotspot" element is supported within the symbol to handle all events (mouseenter etc.).
Touch enabled: in mobile environments just a tap/click listener will be attached (no states are supported).
The parameters of this function allow you to dynamically set the label, icon and clickHandler for the button, so you only need one symbol for different buttons. You can also add a data object, which will be passed to the button instance and will be available in clickHandler callback as second argument.

Example: Creating an animated button

In this example we convert a symbol that already contains the animation for the different states into an animated button. The label, icon and clickHandler will be set dynamically, so that we can define all the button instances in one place. 

          
// Implement Button logic into target symbols
// e.g.
// EC.makeAnimatedButton(sym.getSymbol("MySym"), "Some Label", "images/icon.svg", function() { console.log("CLICKED"); });
//========================================================================================================================
// Create Button 
EC.makeAnimatedButton(sym.getSymbol("MyAnimatedButtonB"), "Button B", "images/icon-01.svg", function() {
//add code for click action here
      window.open("http://www.edgedocks.com", "_blank");	
});

Create static buttons

EC.makeStaticButton();
Convert any given symbol into a static button component with three states (normal, over and down).
Implement the three button states (normal, over, down) as labels on the symbol's timeline. An optional "hotspot" element is supported within the symbol to handle all events (click/touch/mouseenter etc.).
Touch enabled: In mobile environments just a tap/click listener will be attached (no states are supported).
The parameters of this function allow you to dynamically set the label, icon and clickHandler for the button, so you only need one symbol for different buttons. You can also add a data object, which will be passed to the button instance and will be available in the clickHandler callback as second argument.

Example: Creating a static button

In this example we convert a symbol that already contains three button states into a static button. The label, icon and clickHandler will be set dynamically, so that we can define all the button instances in one place. 

          
// Implement Button logic into target symbols
// e.g.
// EC.makeStaticButton(sym.getSymbol("MySym"), "Some Label", "images/icon.svg", function() { console.log("CLICKED"); });
//========================================================================================================================
// Create Button 
EC.makeStaticButton(sym.getSymbol("MyStaticButtonA"), "Button A", "images/icon-01.svg", function() {
		// open alert on button click
        alert("'Button A' was CLICKED");
});

Pause and Unpause a symbol

EC.pause(); | EC.unpause();
With these function you can pause and unpause each and every symbol (timeline) with or without all it's children (optionally recursive). The advantage of EC.pause(...) is that all playback states are stored for proper usage of unpause. The stopAll() and playAll() functions from the official Animate API are not really useful (ALL symbols are started, no matter if they were playing or pausing before), therefore we recommend using the Edge Commons function, which also stores the direction of the current playback.

Set adaptive layouts

EC.setAdaptiveLayouts();
With this feature you can create different symbols that will be displayed on different layout dimensions. Therefore you define an array with the dimensions for alternatvie layouts, hand in a reference to the context (e.g. the stage) and define the (adaptive)container into which the symbols with the different layouts are to be placed. Optional you can add a callback that is being executed on layout switch and/or use the window width as reference. Then the last parameter has to be set to true.

Example: Creating an adaptive header

This sample shows how to create an adaptive header layout with alternative sizes for different devices. That can be easily done with Edge Commons. 

          
// Adaptive Layouts
EC.setAdaptiveLayouts([0, 320, 768, 1200], sym, "content", function(evt) {
	// This code will always be executed whenever the layout changes
}, true);
