Limitations

of working with Edge Animate

Despite all the benefits and advantages Edge Animate has to offer, there are some drawbacks and limitations that need to be mentioned here as well:

Modern Browsers Only
Edge Animate supports all modern browsers from Internet Explorer 9 onward. Older versions will not be able to run your Edge Animate composition. In order to not leave an empty space or cause an error, you have the option to display a (very reduced) substitution called “Downlevel stage”.

File size
Although it no longer requires jQuery, Edge Animate still comes with a comparatively large file size to run in the browser (compared to e.g. a flash animation). Even in simple projects with little or no animation at all, this is too much for some cases, for instance commercial web banners. You should keep this in mind and evaluate if your project might run into trouble regarding download traffic and server sided restrictions. With the latest version Adobe tackled this particular problem, so it might not be so much of a deal anymore.

Performance (DOM manipulation)
The animations created in Edge Animate are basically DOM manipulations – using JavaScript to move elements in the browser window. There are technical limitations regarding performance and speed of such animations.

No support for Canvas/WebGL
Edge Animate only reders to the HTML-DOM. Support for neither Canvas nor WebGL is planned so far. But you can load Canvas content into your Edge Animate compositions with custom JavaScript.

If you keep that in mind, you can avoid running into traps or major problems by choosing appropriate concepts for your projects. It also needs to be mentioned that a great deal of these drawbacks results from the (rather slow) progress regarding web standards. Web standards are a moving target and are constantly developing. It nevertheless seems to be the right direction for modern web technology.
