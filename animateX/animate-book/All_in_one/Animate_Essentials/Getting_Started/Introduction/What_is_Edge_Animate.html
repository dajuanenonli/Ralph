Getting to know Edge Animate

What exactly is Edge Animate?

Let me introduce you to...

Edge Animate: With this tool, Adobe has introduced a multimedia authoring tool that is based on web standards.

A creative game changer

This approach allows you to create animated content that works on all modern browsers and devices. You can use it to create mobile apps and digital publications as well.

The advantage: Although you are working directly on a browser instance, there is no coding knowledge required, since the tool comes along with a visual design surface. This is not only comfortable, but also gives you the necessary precision to stage complex animation choreographies with ease. It is a gateway into a world of compelling web animation and interactive applications.

Edge Animate is part of the Creative Cloud, Adobe's series of tools and services for the creative workflow. As such it is ideal to breathe life into Photoshop compositions and bring Illustrator graphics into the web. It also works well in combination with InDesign for digital publications or with code editors like Dreamweaver.

This guide book will take you through the first steps of getting to know and understanding the fundamental mechanisms of this versatile tool called Edge Animate. It will introduce you to the most important aspects of the interface, teach you how to create your first animation and provide you with the necessary background information to get you started.

Who ought to use Edge Animate and why?

The previous section might already have given you quite an impression on what Edge Animate is about. But who is it actually directed to? Who is supposed to be using it and in which contexts?

Despite the first impression, Edge Animate is not entirely directed at (web-)designers, but takes its place in a creative workflow connecting design and interactivity. Its approach invites the whole range from graphic designers (even with little or no experience in regards to web so far) over motion and UX designers up to creative coder and front end developers.

Edge Animate - Big Picture

Using Edge Animate gives you the possibility to express creativity in terms of web standards without having to write code yourself. You might actually even learn quite a bit about CSS, HTML and JavaScript along the way without really noticing it. You are on the other hand at no time limited to creating visually, since you can as well introduce code manually at every step of the way. You can also use libraries like jQuery UI to enhance your performance. In general, we have experienced that Edge Animate can be easily adapted into pre-existing workflows.

The output of Animate is pure HTML, CSS and JavaScript. That makes it highly interesting for a broad spectrum of use cases, not only the web but e-publications (via Digital Publishing Suite or ePub with Adobe InDesign) and apps as well (i.e. using PhoneGap). In short, it introduces a design centric workflow in a so far rather code driven domain.

Yet there are some rumors and often heard misconceptions that should be addressed here as well:

5 common misconceptions about Edge Animate

1) Edge Animate is yet another banner tool.
Edge Animate is far more than a simple tool for creating web banners. It allows you to stage complex animation choreographies with independent timelines and triggers and create complex layouts. The openness to custom code and external libraries lets you create interactive elements and even entire applications. It is a full-fledged and versatile tool that is applicable for everyday use in real life projects. Frequent update releases via the Creative Cloud add yet more functionality and comfort.

2) It is only for designers, not for a coder.
The tool offers a visual design surface and offers an easy access to webstandards and animation without requiring coding experience. That does not mean that you can not customize the code as well. When doing so, you are not confined to the built-in code view. You can open the project files in any editor of your choice and introduce your custom code. Animate will recognize and accept the altered code, so you can switch back and forth all the time.

3) Animate is the new Flash.
Although there are similarities between the tools, Animate will not be a replacement for Flash. With web standard based animations, Edge Animate is an entirely different approach and tool.

4) You may not use the code produced by Edge Animate.
There is an ongoing dispute of whether code that is generated by tools can be considered "clean" and should be used for "real" projects. We don't want to take part or sides in that discussion here. Let us just point out that the code produced in Edge Animate is JavaScript and encapsulated in seperate files. It is ready for primetime and works across all modern browsers. Plus, you can easily add and alter the code at all time.

5) Edge Animate is for websites (only).
Probably the greatest advantage of the recent shift towards web standards and the mobile sector is that they allow the web to spread and develop into new regions. Interactive and animated content can now be found all over the place, like in e-publications and apps. Animate allows you to create engaging content not only for websites but for pretty much all of these deviations as well.
