Recipe: Creation

Creating simple buttons

with three states

Buttons are very essential aspects of interactive content. If you need a simple button and want to keep it easy, this tutorial is the way to go. Learn how to create a button with three states in order to display normal, hover and clicked states.

Schematic overview of the button:

All visual elements you place into your Animate composition are HTML components. Most of these elements are represented by DIV containers in the DOM. And since it is possible to listen for mouse events (such as “click”) on all DOM containers, we can easily make every element in Edge Animate clickable and therefore create buttons. But it’s often not enough to just add the action to such a button, we also want the button to show its state when the user interacts with it. So we need at least three states: one for the default state, one for hovering over the button and one for holding it down. And since we are working with a tool called “Animate”, we also want to add some smooth transitions.

For the sake of simplicity, this recipe is limited to a hard coded label. There are some more advanced techniques, that allow you to manage several button instances to set the label dynamically. This way you might easily reuse the button. But for now let's stick to the simple solution.

Steps

Steps

Design the default state

Place the button’s base (a simple rectangle) on the stage and add some gradient to it. Also add a solid color that matches the gradient primary color. This is necessary for browsers that do not support gradients (e.g. IE9). Also add a textfield as a label and make it a child of the base container in the elements panel.

Also add a textfield as a label. Make it a child of the base container by placing its layer inside the button base layer in the elements panel.

Convert into symbol

Since we want to create several states of the button we need a decoupled timeline for the button. To achieve this, we simply convert all button elements to a symbol by selecting all parts on the stage and then select Modify > Convert to symbol. Make sure to deselect the “Autoplay timeline” option, since we will control the playhead via script later.

Add labels for the states

With a double click on the new symbol we open the symbol. Now we add three labels on the timeline, “out”, “over” and “down”. We can use a script to jump to these labels later on.

Use a trigger to stop the “over” animation

Now we also add a trigger at the same position where we added the “over” label on the timeline. The script for this trigger is a simple sym.stop() to stop the playhead whenever it reaches this position.

Create the different states

We modify our elements for all three states. We can create a transition from “out” to “over” to specify the transition between the default and the mouseover state. Later we will use the play() and playReverse() function within the event listeners to switch the state. The down state should appear immediately so we don’t need any transitions. So we can disable the “Auto-Transition Mode” before setting the keyframes.

Keep usability in mind

Before we start scripting the behavior of the button we enable the hand cursor for the base element to show the user the interaction option (cursor setting in the property inspector). Make sure that the text is a child of the base so that the cursor settings are applied to the label as well.

Add the button logic

Finally we add the behavior to our component. We use the mediator pattern to reduce the total number of scripts. In this case we add all our behavior logic to the creationComplete event of the button’s symbol: 

          
// Get reference to the symbol's element
var el = sym.getSymbolElement();

// Listen for mouse events
// using .bind() to define the event 
// and a function to add the action
el.bind("mouseenter", (function() {
	sym.play("out");
}));
el.bind("mouseleave", (function() {
	sym.playReverse();
}));
el.bind("mousedown", (function() {
	sym.stop("down");
}));
el.bind("mouseup", (function() {
	sym.stop("out");
}));

Using the buttons

Now you can use your button in your project by simply using the instance you already have at hand on the stage, by dragging a new instance from the library on to the stage or by exporting the button symbol to a “Edge Animate Symbol File” (*.ansym) to be able to reuse the button within another project. To add some behavior to the button use the “click” action on stage level to e.g. open an URL or start playing a timeline.

Further knowledge

Buttons and their labels should not be selectable. To achieve this you can add the following script to your mediator. It simply sets some cross browser CSS properties to make the element with all its children not selectable: 

          
// Make content of button (e.g. text) non-selectable
el.css({"user-select": "none", "-webkit-user-select": "none", "-moz-user-select": "none", "-ms-user-select": "none"});
