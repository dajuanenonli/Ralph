Recipe: Creation

Repeated Background

with CSS

In this recipe you will learn how to place a repeated background image for an element. The image will be placed as tile on click, so that it covers the entire background of the target element. If you want know how to use this CSS property, that is not yet natively supported in Edge Animate, this recipe will explain how you can make it work anyway.

Caution: We are going to need jQuery for this recipe. Since jQuery is no longer automatically integrated within Edge Animate, we have to make sure to include the library for this approach to work.

This is the result:

Background

With JavaScript and jQuery, we are able to modify elements on code-level. We are therefore able to change the html content of a DIV or its CSS properties. Especially the latter can be very useful, for instance when you want to make use of a CSS property that is not yet available in Edge Animate. In our example here we want to include a background-image for a div container that repeats itself automatically. At the moment, this kind of functionality is not supported right out of Edge Animate itself. We can help ourselves here by making use of jQuery and the .css() function to load the background-image when a button gets clicked. By default, the background-image gets placed in the top left corner of the element and repeated both vertically and horizontally.
Note: If you want to change the settings for the repetition of the background-image, you need to set the “background-repeat” property in CSS as well.

Here both states in an overview:

Steps:

Including jQuery

Since we rely on jQuery for this recipe, we need to include it into our project first. You can download jQuery and use it as a local version or import it via URL (CDN) in the scripts panel:

Creating the target container

We start by creating the element that the background-image will be loaded into. In this case, we use a simple rectangle with rounded corners and place it onto the stage. Then we change its name to “box”, so that it is easier to refer to this specific element on code level later on.

Including the background-image

The intended background consists of single tiles that get repeated in both directions. The advantage is that we only need a small image to cover a big area. This is also handy in terms of flexible layouts, since a repeated background-image automatically covers the entire background, regardless of its actual size.

For this recipe, we have already prepared the tile that we are going to use:

We simply copy the image into the images folder of our Animate composition and remember its name (‘background.png’). Tip: If you drag the image file into Animate and onto the stage, you can remove it again and it still gets saved into the images library of the project.

Preparing the button

Now we want to create a button that allows us to trigger the event to change the css of the target div. We use a rectangle and a nested text element to construct our button.

Then we add a click event to this button via "Open Actions..." (If you want to know more about buttons, check out the recipe on simple buttons). In this click event, we can now define the appropriate function.

Defining the function

First we need a reference to the target element. So we use jQuery and the name of the element, which we previously had set to “box” for exactly this reason. Then we can use jQuery's .css() method, which either expects two parameters (the property and the new value) or an object with several properties.

In our case here, we need to set both the “background-image” and the "background-size" to prevent it from scaling (which would prohibit the intended tile effect). So we use curly braces to create said object instantly and write down the property / value pairs. The value for "background-image" is the location of the background-image. Since we already included the image tile into the images folder of our composition, we can use the term “url(images/background.png)” to get the direction of it. The value for "background-size" ought to be "auto".

Note that both property and value need to be stated in quotes and are seperated by a colon.
The entire function therefore is as follows: 

          
sym.$("box").css({"background-image": "url(images/background.png)", "background-size": "auto"});

And that’s it. You can now start the composition in the browser and see how the background-image gets loaded as soon as you click the button.

You can use this technique to change other css properties as well, which can be very helpful in some occasions. You can experiment with it and also use different event types, if you like.
