Creating a speech bubble

with plain elements

This very primitive recipe will show you how you can create a speech bubble with plain elements in Edge Animate. It's nothing fancy, but usable in many occasions, and a good way to start practicing our creation skills in Animate. So let's get to work!

Result

Result

If you combine several elements of the same color and apply transformations, you can create all sorts of shapes out of plain elements. The advantage is that you don't need to import external assets and can work with plain HTML elements.

Challenge: Try to create a simple speech bubble by combining two rectangles. This is what the speech bubble is supposed to look like:

Have you already figured it out? Well, let's take it one step at a time:

First you create the larger rectangle as base. In this example it is one with rounded corners. Then you draw a second rectangle in the shape of a square underneath that first element. Make sure to use the same color for both elements so that they fit into each other.

Now select the smaller rectangle and go over to the Transform section in the elements panel. Use the rotate property to turn the elements by 45 degrees. Then move the element above the larger rectangle, so that it overlaps for the most part. This will add a triangular shape to the large rectangle. Voilà...

As soon as you de-select the elements it seems as if it is only one shape, that looks like a speech bubble.

Tipp: If you want to make sure that both elements behave like one element together, move over to the Elements panel and drag the smaller rectangle upon the other one. This way it will be placed within the larger element in regards to the DOM structure. Therefore they form a group and adopt animations for both elements at the same time.

Creating simple buttons

with three states

Buttons are very essential aspects of interactive content. If you need a simple button and want to keep it easy, this tutorial is the way to go. Learn how to create a button with three states in order to display normal, hover and clicked states.

Schematic overview of the button:

All visual elements you place into your Animate composition are HTML components. Most of these elements are represented by DIV containers in the DOM. And since it is possible to listen for mouse events (such as “click”) on all DOM containers, we can easily make every element in Edge Animate clickable and therefore create buttons. But it’s often not enough to just add the action to such a button, we also want the button to show its state when the user interacts with it. So we need at least three states: one for the default state, one for hovering over the button and one for holding it down. And since we are working with a tool called “Animate”, we also want to add some smooth transitions.

For the sake of simplicity, this recipe is limited to a hard coded label. There are some more advanced techniques, that allow you to manage several button instances to set the label dynamically. This way you might easily reuse the button. But for now let's stick to the simple solution.

Result

Result

Steps

Steps

Design the default state

Place the button’s base (a simple rectangle) on the stage and add some gradient to it. Also add a solid color that matches the gradient primary color. This is necessary for browsers that do not support gradients (e.g. IE9). Also add a textfield as a label and make it a child of the base container in the elements panel.

Also add a textfield as a label. Make it a child of the base container by placing its layer inside the button base layer in the elements panel.

Convert into symbol

Since we want to create several states of the button we need a decoupled timeline for the button. To achieve this, we simply convert all button elements to a symbol by selecting all parts on the stage and then select Modify > Convert to symbol. Make sure to deselect the “Autoplay timeline” option, since we will control the playhead via script later.

Add labels for the states

With a double click on the new symbol we open the symbol. Now we add three labels on the timeline, “out”, “over” and “down”. We can use a script to jump to these labels later on.

Use a trigger to stop the “over” animation

Now we also add a trigger at the same position where we added the “over” label on the timeline. The script for this trigger is a simple sym.stop() to stop the playhead whenever it reaches this position.

Create the different states

We modify our elements for all three states. We can create a transition from “out” to “over” to specify the transition between the default and the mouseover state. Later we will use the play() and playReverse() function within the event listeners to switch the state. The down state should appear immediately so we don’t need any transitions. So we can disable the “Auto-Transition Mode” before setting the keyframes.

Keep usability in mind

Before we start scripting the behavior of the button we enable the hand cursor for the base element to show the user the interaction option (cursor setting in the property inspector). Make sure that the text is a child of the base so that the cursor settings are applied to the label as well.

Add the button logic

Finally we add the behavior to our component. We use the mediator pattern to reduce the total number of scripts. In this case we add all our behavior logic to the creationComplete event of the button’s symbol: 

          
// Get reference to the symbol's element
var el = sym.getSymbolElement();

// Listen for mouse events
// using .bind() to define the event 
// and a function to add the action
el.bind("mouseenter", (function() {
	sym.play("out");
}));
el.bind("mouseleave", (function() {
	sym.playReverse();
}));
el.bind("mousedown", (function() {
	sym.stop("down");
}));
el.bind("mouseup", (function() {
	sym.stop("out");
}));

Using the buttons

Now you can use your button in your project by simply using the instance you already have at hand on the stage, by dragging a new instance from the library on to the stage or by exporting the button symbol to a “Edge Animate Symbol File” (*.ansym) to be able to reuse the button within another project. To add some behavior to the button use the “click” action on stage level to e.g. open an URL or start playing a timeline.

Further knowledge

Buttons and their labels should not be selectable. To achieve this you can add the following script to your mediator. It simply sets some cross browser CSS properties to make the element with all its children not selectable: 

          
// Make content of button (e.g. text) non-selectable
el.css({"user-select": "none", "-webkit-user-select": "none", "-moz-user-select": "none", "-ms-user-select": "none"});

Sexy Badges

with CSS

This recipe will show you a simple way to make use of CSS class selectors to define CSS properties outside of your Animate project. We are going to use three customized text fields to create a colored label and two badges using only text elements and an additional CSS declaration in the head section of the html document of the Animate project.

Background

Edge Animate is a great tool when it comes to creating, designing and animating with CSS. You can yet achieve even more when you start using class selectors to enhance your design options with additional CSS. This then allows you to build and use CSS templates, rely on existing CSS code from external sources and to use your entire CSS knowledge to create even more awesome Animate compositions.

Steps:

Creating the text fields

We start by creating plain text elements in Edge Animate. We want to create three different badges in this tutorial, so we use the text tool to generate three text fields on the stage. Then we type in the text for each badge accordingly.

We can of course use the property panel in Animate to change the text color and font type for the basic settings of the badges, so that we don’t have to define all the CSS properties manually later in the additional style.

Defining the CSS classes

Now we want to define class selectors for the badges, one for each kind. To do this, we select the text field for the appropriate badge and click on the button to set a class selector (the tiny “c” next to the “open actions” braces).

We start here with the text field at the top. For CSS classes it is always a good choice to use names that indicate the purpose of the class. So in this case we use ‘color-label’, since we only want to add background color to the text-div to turn it into a label. The second text-element that we want to customize gets the class ‘color-badge’ and the third one ‘badge3d’.

Adding custom CSS to the html file

The next part requires a few lines of code. First we open the html file of the Animate project in an editor of our choice. I for my part like to use Brackets for example. We now want to define the CSS styles for the single classes right in the html document. Therefore we add a new <STYLE> tag at the end of the <HEAD> section. Within this new style tag we add the CSS for the single classes .color-label , .color-badge and .badge3d.

In case of the first class we only need to define padding and background-color to create our colored label:

Repeated Background

with CSS

In this recipe you will learn how to place a repeated background image for an element. The image will be placed as tile on click, so that it covers the entire background of the target element. If you want know how to use this CSS property, that is not yet natively supported in Edge Animate, this recipe will explain how you can make it work anyway.

Caution: We are going to need jQuery for this recipe. Since jQuery is no longer automatically integrated within Edge Animate, we have to make sure to include the library for this approach to work.

This is the result:

Background

With JavaScript and jQuery, we are able to modify elements on code-level. We are therefore able to change the html content of a DIV or its CSS properties. Especially the latter can be very useful, for instance when you want to make use of a CSS property that is not yet available in Edge Animate. In our example here we want to include a background-image for a div container that repeats itself automatically. At the moment, this kind of functionality is not supported right out of Edge Animate itself. We can help ourselves here by making use of jQuery and the .css() function to load the background-image when a button gets clicked. By default, the background-image gets placed in the top left corner of the element and repeated both vertically and horizontally.
Note: If you want to change the settings for the repetition of the background-image, you need to set the “background-repeat” property in CSS as well.

Here both states in an overview:

Steps:

Including jQuery

Since we rely on jQuery for this recipe, we need to include it into our project first. You can download jQuery and use it as a local version or import it via URL (CDN) in the scripts panel:

Creating the target container

We start by creating the element that the background-image will be loaded into. In this case, we use a simple rectangle with rounded corners and place it onto the stage. Then we change its name to “box”, so that it is easier to refer to this specific element on code level later on.

Including the background-image

The intended background consists of single tiles that get repeated in both directions. The advantage is that we only need a small image to cover a big area. This is also handy in terms of flexible layouts, since a repeated background-image automatically covers the entire background, regardless of its actual size.

For this recipe, we have already prepared the tile that we are going to use:

We simply copy the image into the images folder of our Animate composition and remember its name (‘background.png’). Tip: If you drag the image file into Animate and onto the stage, you can remove it again and it still gets saved into the images library of the project.

Preparing the button

Now we want to create a button that allows us to trigger the event to change the css of the target div. We use a rectangle and a nested text element to construct our button.

Then we add a click event to this button via "Open Actions..." (If you want to know more about buttons, check out the recipe on simple buttons). In this click event, we can now define the appropriate function.

Defining the function

First we need a reference to the target element. So we use jQuery and the name of the element, which we previously had set to “box” for exactly this reason. Then we can use jQuery's .css() method, which either expects two parameters (the property and the new value) or an object with several properties.

In our case here, we need to set both the “background-image” and the "background-size" to prevent it from scaling (which would prohibit the intended tile effect). So we use curly braces to create said object instantly and write down the property / value pairs. The value for "background-image" is the location of the background-image. Since we already included the image tile into the images folder of our composition, we can use the term “url(images/background.png)” to get the direction of it. The value for "background-size" ought to be "auto".

Note that both property and value need to be stated in quotes and are seperated by a colon.
The entire function therefore is as follows: 

          
sym.$("box").css({"background-image": "url(images/background.png)", "background-size": "auto"});

And that’s it. You can now start the composition in the browser and see how the background-image gets loaded as soon as you click the button.

You can use this technique to change other css properties as well, which can be very helpful in some occasions. You can experiment with it and also use different event types, if you like.

Google Maps

Integration of an external source

In this recipe you will learn how to integrate Google Maps into your Animate composition. So if you want to integrate a customized map into your project, e.g. to point out a specific location or route, this recipe will lead you the way.

This is the result:

Background

Integrating a generic map into your website really is not big of a deal thanks to Google Maps and the embed code it provides. You see it all the time. Sometimes however you would prefer to customize the map and point out a specific location, route or add further information. Now Google came up with a new option here, called “Maps Engine Lite”, which at the time of writing is still in its beta stage, so there might be some changes in the future. Still, since it already has some nice aspects to it, it’s worth taking a closer look.
So let’s see how we can use it to built a custom map into an Animate project, to point out the location of our favorite sample restaurant “Murasaki Sushi”.

Steps:

Designing the Animate project

We use Edge Animate to create a nice little surrounding page for our map. Basically, it simply consists of three artworks, a background with gradient and two white rectangles with rounded corners. Of course you can design it any way you want or integrate the map into an existing project.

Creating the container for the map

Now we draw a rectangle on top of the white rectangle to create a container for the map. Rectangles in Edge Animate are basically divs in the DOM, so that we will be able to inject any html code with a simple JavaScript function. As you will see, Google will provide us with an IFrame later on, which we then can inject into this div. Make sure to rename this rectangle with “map”, so we can refer to it on code level with this ID.

Another advantage here is, that we can determine the size of the contained map with the measurement of the rectangle. We don’t need to worry about the color, since the rectangle itself will be replaced by the map. If you drag this container rectangle onto the other rectangle in the elements panel, you can group them together, which makes it easier for animations or repositioning.

Customizing the map

We can now move on to customizing the map for our purposes. Therefore we go to mapsengine.google.com and start editing the map by adding markers and labels as well as changing the color scheme

We can also paste in details and additional information for the location here.

Making the map publicly accessible

We can only embed the map into a website if we make it public first. Therefore we have to use the ‘share’ button on the upper right corner to access the sharing settings for the map. Change the visibility options to “Public on the web” and save the settings.

Getting the embed code for the map

Now that we have made the map accessible, we get access to the embed code in the map menu (the folder icon on the upper left) via the option “Embed on my site”.

We copy the entire code-snipped, i.e. the iframe that contains the link to our custom map so that we can use it within our Animate project.

Embedding the map into Animate

So let’s get back to Edge Animate and our project. We need to implement a few lines of code now to get the map loaded into our DIV container, so we add an action for creationComplete for the stage. Again, what we want to achieve now is to get a reference to the target element, which is the DIV with the ID ‘map’, and inject it with the IFrame of our custom map. We set and use the variables “map” and “mapCode” to make it easier to follow here. But let’s take it step by step:

First we define a variable that saves the jQuery reference to the target element: 

          
.var map = sym.$(“map”);

Then we use another variable to save the IFrame code that we copied from our map. Since we want the map to adapt to the size of the containing div element, we need to make some minor adjustments concerning the width and the height within the iframe itself. So we exchange the according properties with ‘+map.width()+’ and ‘+map.height()+’ respectively. Note that we have to use both single and double quotation marks so that they don’t interfere each other. The code looks like this: 

          
var mapCode = '<iframe src="http://mapsengine.google.com/map/embed?mid=zsKqVOZ51FEQ.kupWm_JOuHgo" ' +'width="'+map.width()+'" height="'+map.height()+'" style="border: none;"></iframe>';

Now that we have both variables defined, we can use them to add the generated map code into the map container, using the JavaScript method .html(): 

          
map.html(mapCode);

So combined the code block reads as follows: 

          
// (1) Get the target element for the map
var map = sym.$("map");
// (2) Use '' to not get a conflict with the provided code that includes ""
// (3) Modify the width and height to be more flexible and add style to get rid of the ugly border
var mapCode = '<iframe src="http://mapsengine.google.com/map/embed?mid=zsKqVOZ51FEQ.kupWm_JOuHgo" '
	 +'width="'+map.width()+'" height="'+map.height()+'" style="border: none;"></iframe>';
// (4) Add generated map code to the map container
map.html(mapCode);

That’s it. When you open the project in the browser, you will now see that the custom map gets loaded into the div container and can be used just like any other integration of Google Maps.

Publish to Wordpress

Integrating a composition in a Content Management System.

In this recipe you will learn how to publish your Animate composition in Wordpress. This way you can use and publish your project directly in a blog entry to create even more awesome, interactive and animated content. We will be using the Animate Plug-In for Wordpress written by our colleague Timm Jansen.

Background

In this exercise we want to publish an existing Animate composition in a Wordpress environment. We will be using the Animate Deployment Package for the composition and the Edge Suite template for Wordpress to get this done. Wordpress is a common web content management system for creating and maintain microsites such as blogs. It is less complex than Drupal or Typo3, but at the same time offers a broad variety of templates to choose from. It is a good choice if you want to publish content to the web without having to worry much about backend development (FYI: There is a Drupal solution for Animate available as well).

Note: Wordpress requires a webserver to be installed on. If you don’t have access to one yourself, you can use one of the numerous offers on the web for free webspace or run a local webserver on your device. You will also find recommendations in the helpful tutorials about installing Wordpress. Check them out if you run into any trouble. This recipe however is based on the assumption that you already have an existing Wordpress page up and running. So let’s get started...

Here's the result:

Steps:

Creating an Animate Deployment Package

This recipe works with any kind of Animate project. And as we want to focus on the publishing of the composition, we can simply revert to one of the recent projects we created in the other recipes so far instead of creating a completely new one. So lets get back to our favorite sushi restaurant and the Google Map recipe (if you haven’t done that recipe yet, you can just take any other Animate project of yours – or use the index.oam file provided in the projects folder.

»Get the .OAM file«

You find the corresponding file in the projects folder at
projets/020 Recipes/060 PublishToWordpress

When we save a project in Edge Animate, it automatically creates the file structure that we already encountered several times by now. But apart from that, we also have a number of options to generate different types of files with the publish function in Edge Animate. The standard setting here is ‘web’, but in our case we want to create the Animate Deployment Package (*.oam), a generic type of file designated for Animate projects, which contains all the relevant information in one single file. This file can be interpreted later by some other Adobe tools like Dreamweaver, Muse and Indesign or the CMS Plug-In “Edge Suite for Wordpress” which we are going to use in this recipe.
In order to create this .oam file, you need to open “Publish Settings” in the File menu in Edge Animate first. There you can select “Animate Deployment Package” on the left hand side. You can change the target directory or the name of the .oam file if you want to, but for now we leave everything as it is. We only deselect the option “Web” on the left hand side.

Caution: You have to click on the “Publish” button to actually create the file. If you click on “Save” it will only save the settings and you will still have to publish it (“File” > “Publish”).
If you check your Animate directory, you should now find a new folder called “publish” containing another folder (“animate package”) with the .oam file.

Installing the Edge Suite Plug-In for Wordpress

There are a few preparations that need to made in Wordpress before we can load our Animate composition into a new blog entry. First of all, we need to install a Plug-In called “Edge Suite”, crafted by our colleague and friend Timm Jansen. This extension will be able to handle the *.oam file and allows you to integrate it into your Wordpress site.

To install it, you have to open the plug-in section on your Wordpress dashboard, select "Add new" and search for "Edge Suite". Use the "install now" option and follow the instructions.

Loading the Animate Deployment Package with the Edge Suite

After the installation the Edge Suite will appear as a new item in your Wordpress dashboard, right below the settings. In this section you can browse for the Animate Package to upload it to Wordpress..

Enabling shortcodes

After successfully uploading the Animate Deployment Package, our composition is already integrated within the files structure of our Wordpress site. There are several options to place Animate compositions, but we want to stick to simple blog entries for now. The easiest way to do so is by using shortcodes. These have to be enabled under “Edge Suite settings” first. Therefore we select the checkbox next to “Widget shortcode” to activate them.

These shortcodes make it really easy to use uploaded Animate compositions in the content area of your Wordpress site, for instance as a blog post. The shortcode itself looks like this: [edge_animation id=””]

Finding out the ID of the Animate composition

As you can see, the shortcode requires an ID from the Animate project. So what is this ID and where can we find it? This is a bit complicated. Wordpress automatically references uploaded Animate packages with a continuous numbering, so that each composition gets its unique number, which can then be used in the shortcode.

To find out this number, we have to go back to the ‘manage’ page of the Edge Suite. There you will find a drop-down menu under “Delete selected composition”. Opening that menu will show you all the available compositions with the equivalent composition number. But be careful not to accidentally delete the composition! Just remember the number. In this case the composition number is “7”.

Using the shortcode in a new post

Now we can edit and existing entry or add a new post to our wordpress page. To implement our composition, we simply add the shortcode with the composition number at the end of our blog text:.

When we are done editing the post we can publish it and take a look at our wordpress page. And there it is, our animate composition in wordpress, neat and running.
