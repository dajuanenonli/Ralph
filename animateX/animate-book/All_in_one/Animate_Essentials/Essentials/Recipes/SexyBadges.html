Recipe: Creation

Sexy Badges

with CSS

This recipe will show you a simple way to make use of CSS class selectors to define CSS properties outside of your Animate project. We are going to use three customized text fields to create a colored label and two badges using only text elements and an additional CSS declaration in the head section of the html document of the Animate project.

Background

Edge Animate is a great tool when it comes to creating, designing and animating with CSS. You can yet achieve even more when you start using class selectors to enhance your design options with additional CSS. This then allows you to build and use CSS templates, rely on existing CSS code from external sources and to use your entire CSS knowledge to create even more awesome Animate compositions.

This is what the final badges look like:

Steps:

Creating the text fields

We start by creating plain text elements in Edge Animate. We want to create three different badges in this tutorial, so we use the text tool to generate three text fields on the stage. Then we type in the text for each badge accordingly.

We can of course use the property panel in Animate to change the text color and font type for the basic settings of the badges, so that we don’t have to define all the CSS properties manually later in the additional style.

Defining the CSS classes

Now we want to define class selectors for the badges, one for each kind. To do this, we select the text field for the appropriate badge and click on the button to set a class selector (the tiny “c” next to the “open actions” braces).

We start here with the text field at the top. For CSS classes it is always a good choice to use names that indicate the purpose of the class. So in this case we use ‘color-label’, since we only want to add background color to the text-div to turn it into a label. The second text-element that we want to customize gets the class ‘color-badge’ and the third one ‘badge3d’.

Adding custom CSS to the html file

The next part requires a few lines of code. First we open the html file of the Animate project in an editor of our choice. I for my part like to use Brackets for example. We now want to define the CSS styles for the single classes right in the html document. Therefore we add a new <STYLE> tag at the end of the <HEAD> section. Within this new style tag we add the CSS for the single classes .color-label , .color-badge and .badge3d.

In case of the first class we only need to define padding and background-color to create our colored label: 

          
.color-label {
      padding: 14px 20px;
      background-color: rgba(177,226,111,1.00);
}

The second class is meant to define a badge with two rounded corners and a gradient. This requires some more properties, so we have to set padding, border-radius, background-color and also background-image for the gradient. Note that we need browser prefixes here: 

          
.color-badge {
      padding: 14px 20px;
      border-radius: 16px 0 16px 0;
      background-color: rgba(52,207,194,1.00);
      background-image: linear-gradient(rgb(0, 216, 203) 0%, rgb(23, 144, 137) 100%);
      background-image: -webkit-linear-gradient(rgb(0, 216, 203) 0%, rgb(23, 144, 137) 100%);
 }

The third class is meant to create a 3D badge and is inspired by codrops. It uses a variety of CSS properties to let the badge appear to be three dimensional. So we have to define quite a few properties to get this result. Most of the declarations are due to browser prefixes, however. What we need to define here is background-color, padding, border-left, margin-left, background-image, border-radius and box-shadow: 

          
.badge3d {
        background-color:#3bb3e0;
        padding:10px 20px;
        border-left:solid 1px #2ab7ec;
        margin-left:35px;
        background-image: linear-gradient(bottom, rgb(44,160,202) 0%, rgb(118, 218, 255) 100%);
        background-image: -o-linear-gradient(bottom, rgb(44,160,202) 0%, rgb(118, 218, 255) 100%);
        background-image: -moz-linear-gradient(bottom, rgb(44,160,202) 0%, rgb(118, 218, 255) 100%);
        background-image: -webkit-linear-gradient(bottom, rgb(44,160,202) 0%, rgb(118, 218, 255) 100%);
        background-image: -ms-linear-gradient(bottom, rgb(44,160,202) 0%, rgb(118, 218, 255) 100%);
        background-image: -webkit-gradient(linear, left bottom, left top, color-stop(0, rgb(44,160,202)), color-stop(1, rgb(118, 218, 255)));
        -webkit-border-radius: 5px;
        -moz-border-radius: 5px;
        border-radius: 5px;
        -webkit-box-shadow: inset 0px 1px 0px #2ab7ec, 0px 5px 0px 0px #156785, 0px 10px 5px #999;
        -moz-box-shadow: inset 0px 1px 0px #2ab7ec, 0px 5px 0px 0px #156785, 0px 10px 5px #999;
        -o-box-shadow: inset 0px 1px 0px #2ab7ec, 0px 5px 0px 0px #156785, 0px 10px 5px #999;
        box-shadow: inset 0px 1px 0px #2ab7ec, 0px 5px 0px 0px #156785, 0px 10px 5px #999;
}

Back to Animate

After saving the altered html file we can return to Edge Animate. Here we get informed that the code was changed outside of Animate and asked if we want to apply those changes. When we agree we see that the text fields now look according to our CSS styles. We can now proceed and create an Animate composition with our badges. We can also use the defined CSS classes to create more badges of the same kinds in the project by creating new text elements and assigning the according class to them - which comes in very handy when you have to use the same kinds over and over again.

That's all there is to it. The advantage is, that Animate renders the CSS we have included as well, so we see the result in the design preview already. This leaves us in full control over the visual aspects of our composition. Now it is up to you to create awesome projects with it. Enjoy!
