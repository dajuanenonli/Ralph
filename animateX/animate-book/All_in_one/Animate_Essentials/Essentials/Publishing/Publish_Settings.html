Publishing Animate Compositions

Publish Settings

Getting it right.

In this section you will get to know the different options for publishing your Animate project (Web, Animate Deployment Package and iBooks / OS X), and discover what's behind these options. Learn how to make the right settings in the publishing dialog and get to know about the files and structures you create here.

Publish Settings: Dialog

Before actually publishing your Animate project, you should take a look at the publish settings. These settings are available through the publish settings dialog. Here you can select the context (web, other tools, widget) and get access to further options like CDN hosting.

You can access the publish settings through the main menu. Go to "File > Publish Settings..." to open up the corresponding dialog.

The different contexts are available through the checkboxes on the left hand side under Publish Target. You can select more than one target at a time. The options available in the dialog vary from target to target, so that you have different settings for web, deployment package and iBooks / OS X widgets.

Save and Publish

Keep in mind that this dialog enables you to set your publish settings. When you hit "save", you only save the settings you made. To actually publish your project according to these settings, you have to click on "publish" on the bottom left. This is unfortunately in fact a bit misleading.

Hint: Once you have made and saved your publish settings, you don't have to go back to the dialog to publish it. You can simply go to "File > Publish".

Web and optimized HTML

The first and probably most frequently used target option is to publish for the web and as optimized HTML, so let's take a closer look at it here.

Although the Animate composition as it is is already able to run in a browser, the project should be published before used in the final web and HTML target. That is because your working project is not yet optimized and contains for example the AN file, which you don't need in the published project. Also you can make useful settings and adjustments, which we will quickly walk through here.

Target Directory

Select the target directory for your published project. By default it is published in an new folder called "publish/web" in the same directory as your current project. But you can choose any location you like.

Asset organization

Since Edge Animate is managing the assets included in your project like images, videos and scripts, it will include them into the published project as well. In this section you can define the name of the corresponding folders for each file type to make it fit to your targets' usual structure.

Host runtime files on Adobe CDN

With this option enabled, the published project will download and use the Edge Animate runtime that is hosted on a CDN server, which will allow your content to download faster. This means that an Internet connection is required for the composition to run. Uncheck the option if your files need to run without an Internet connection or if you are using your own hosting.

Publish content as static HTML

By default, the entire HTML structure of your composition gets injected at runtime via JavaScript. With this option enabled, the content will already be placed as static HTML, so that the output already resambles the composition structure and make the text elements indexable.

Animate Deployment Package

The Animate Deployment Package is a special format that packs your project into a single file (.OAM) and allows you to import and use your Animate composition in other tools like Dreamweaver and Muse. Thanks to a plug-in written by Timm Jansen, you can also use this format to easily include your Animate composition into Wordpress and Drupal.

Target Directory

As with web / HTML published files, you can set the target directory for the Animate Deployment Package in the publish settings.

Published Name

Since the published package will be only be a single file, you can set the name here. By default, the .OAM file will have the same name as the html file of your project.

Poster Image

Here you can set an image that will be displayed when the package is placed inside of other tools.

Host runtime files on Adobe CDN

This option has the same effect as with the web / HTML target option. When enabled, the composition will use the CDN version of the Edge Animate runtime, resulting in smaller file size and faster downloads but making an Internet connection necessary.

iBooks / OS X

The third option to publish an Animate project is to create a .wdgt file, allowing you to use the composition for iBooks or as dashboard widget on OS X.

The stettings here are similar to the ones we already discussed: You can select a Target directory, define a Name for the published project and select a Poster Image.

...and Publish!

Once you have set your publish settings, you can finally hit the "Publish" button in the dialog or choose "File > Publish" in the menu. The appropriate folders will then being created and can then be used in the desired context.
