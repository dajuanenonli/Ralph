Publishing Animate Compositions

Exporting and publishing

Getting it 'out there'

This chapter will show you how you can publish your Edge Animate projects and what you should consider for the different options. You will learn therefore learn which settings you should use for each purpose. Be it exporting for web, as widget or as special deployment package for use in Muse, Dreamweaver or Wordpress/Drupal - this chapter will guide you the way.

Why publish the project?

There are a couple of reasons to actually publish your Animate project after you have finished it and to not use the project files as such, although they already work in a web environment. The main reason here is that the working project is not optimized before it is published. Also, publishing allows you to make important settings like enabling or disabling CDN hosting.

What is CDN hosting?

All Animate compositions have one thing in common: They rely on the Edge Animate runtime to work. This runtime is the core that provides the functionality and API. Since this foundation is the same for all Animate compositions, you don't need to carry it around all the time. This is where the CDN servers come into play: They provide the runtime at the nearest location, so that the user gets it as fast as possible. The upside is a smaller file size (on your server) and a faster download for the user. The disadvantage is that an Internet connection is required, so that your composition will only work when the user is online. Therefore you have to choose wisely. If in doubt, disable this option so that the runtime will be provided along with the composition.
