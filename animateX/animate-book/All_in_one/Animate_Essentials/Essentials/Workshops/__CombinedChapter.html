Very first steps

Welcome to your first practical experience with Edge Animate. Before we go further into detailed information about the tool and its functions, we want to give you a quick kickstart of working with the tool and a hands-on introduction. In this guided tutorial you will create your first animation, add a link to an external website and learn how you can run the project in a browser. Plain and simple. Step by step tutorial

Script

  1. Start by opening the project folder of this tutorial. It already consists a prepared Animate project. Simply double click on the AN file to open the project in Edge Animate.
  2. The composition already contains the neccessary assets in the images folder. Yet the stage is still empty as there are no assets placed. Go to the right hand side of the interface. There you will find the Library, that contains the items in the “Images” section.
  3. First of all, place the background image by dragging it onto the stage.
  4. Then search for the item “ATable.png”. Drag it onto the stage and place it fairly into the center.
  5. Go back to the Library and look for “Hotspot.png”. Place this item on top of the stage, again simply with drag and drop.
  6. Now look for the Lady.png and place it on the left side in the grey area beside the stage. This way we can use an animation to let her appear from the off. And here’s how we do it:
    1. On the left side, you see the property inspector. Make sure that the Lady is selected (blue box around the image, otherwise click on it once again). Look for the X in the property inspector and click on the small diamond shaped button next to it. This will create a keyframe for the position of the image on the X-Achsis.
    2. At the bottom, you see the timeline with a playhead. Click on the playhead (the yellow handle above the red line), and drag it to approximately 1 second (0:01).
    3. Now hold the shift key pressed and drag the Lady image to the right side of the table. You will see that an animation track (colored bar) will automatically appear on the timeline.
    4. Press space bar to watch this animation take place.
  7. Finally, we want to use the Hotspot image to serve as a link to an external website.
    1. Therefore, right-click on the Hotspot.png image.
    2. In the menu that opens up, look for ‘Open actions for “Hotspot”’ and select it.
    3. Now a code panel opens up. First of all, we have to choose an event we want to react to. In this case we select "click". Now we can use JavaScript to define what should happen, when the user clicks on the hotspot. Fortunately, we can use predefined snippets for our purpose:
      1. Click on the option “Links”, and then select “Link in Same Window”. Notice how the necessary code is automatically added in the code view above. Here you can change the URL to your preference, e.g. edgedocks.com, or simply leave it as it is (adobe.com). You can then close the code window, the script is stored automatically and needs no further confirmation.
    4. Finally, we want to indicate that the hotspot is an interactive item by changing the mouse cursor to pointer. Select the hotspot image and click on the button in the cursor section of the property inspector. Now click on the pointer (the hand icon) and hit the button again to close the selection.
  8. Almost done. It is now time to save the project. Go to the top menu, select "File" and use “Save as” to save the project in an appropriate location.
  9. Last but not least we want to see the result in a real browser. Therefore open the File panel in the menu again and select “Preview In Browser”. The composition will now be launched in your default browser.

Voilá, you have just created your first, interactive Animate composition. Congratulations!

Interactive Storytelling

Sprite sheets and scenes

This workshop is a bit more advanced and will teach you how to use Edge Animate for interactive story telling. It combines several aspects of working with Edge Animate: You will learn how to create and control a sprite sheet animation, how to build an animated scenery with a visual parallax effect and how to add interactions with a spotlight overlay. For this last part, we will rely on the Edge Commons extension library, so you will learn how to integrate and use it in an actual project. Here we go!

Step by step tutorial

Script

Start off your project with a couple of preparations:

  1. First, set the size of the stage to 600px by 400px and save your project in an appropriate folder.
  2. Now import the images from the assets folder into your project by copying the subfolders into the images folder of the created project.
  3. Place the background image onto the stage, so that it fits the left side. It is supposed to be three times larger, so it reaches out of the stage on the right side.
  4. Make sure that overflow is set to “hidden” for the stage, so that this additional background is not visible in the browser.
  5. Use the tools to create a rectangle above the stage with the same width and call it “scenes”. This will serve as container for all the assets so that we can move them all at once.
  6. Go to the elements panel and place the background image into this scene container.

Now we can take care of our first scene:

  1. Create a simple animation: Let the speech bubble appear from above.
    1. Place the speech bubble image and the first steps text above the stage in the center.
    2. Select both assets, right click on them and use “Group Elements into DIV” to merge them. Make sure they are placed in the “scenes” container.
    3. Set the playhead to approximately 3 seconds and activate the pin tool.
    4. Move the playhead 1 second further.
    5. Create an animation by moving the speech bubble down and onto the stage.
    6. Select the created animation track and add the easing “Ease Out: Back”.
  2. Create a sprite sheet animation
    1. Use the rectangle tool to create a new element (100px by 133px)
    2. Right click on the element and select “Convert to Symbol...”
    3. Change the name of the symbol to “character” and remove the rectangle to get an empty container.
    4. Now place the sprite sheet image into this container, so that the first image fits its borders.
    5. Set a keyframe for the Left position (X-Axis).
    6. Disable “Auto-Transition Mode”
    7. Activate the grid for the timeline and set it to 10/second.
    8. Move the playhead one position further on the grid, move the sprite sheet 100px to the left (-100px on X-Achsis) and create a new keyframe (if not created automatically). Repeat this step for every image of the sprite sheet.
    9. Set overflow to hidden for the symbol, so that only one image is visible at a time.
    10. Create a walking loop
      1. Set a label with the value “walk” a the second keyframe (first instance of the walking movement).
      2. Move the last 4 keyframes one step further on the grid to get an empty space at the end of the walking movement
      3. Place a trigger in that very space and enter the action for playing from the label on the timeline: sym.play(“walk”);
    11. Now you can preview the result in the browser. The character should be walking on the spot.
  3. Go back to the stage scope and animate the character onto stage and under the speech bubble, using the pin tool.
  4. Letting the character talk:
    1. Go back into the symbol and place a label above the fourth to last keyframe (where the character first faces the front). Enter “talk” as label tag.
    2. Move the second last keyframes again one step further on the timeline grid.
    3. Place a trigger in the gap, to create another loop: sym.play(“talk”);
    4. Now go back to the stage level and add a playback command (“+” icon next to the character symbol in the timeline control panel) at the end of the character animation (when he reaches the speech bubble). Choose “Play from”, then switch to labels and select “talk”.
  5. Animate the entire scene and resume walking
    1. Move the playhead to the right to determine the duration of the character talking.
    2. Activate the pin and move the playhead appr. 6 seconds to the right.
    3. Select the “scenes” element and move it 1200px to the left, so that the background now fits the right side of the stage.
    4. Move the character to the right so that he remains visible on the stage and ends up in the third scene.
    5. Use the playback commands to resume the walking cycle by selecting “Play from” -> “walk” at the beginning of the animation.

Creating a visual parallax effect and animation loop

  1. To create the second scene, we place all relevant assets (apart from the fish) and distribute them, so that the cloud is in the back and desktop and mobile device are in front of the big screen.
  2. Now select the laptop and convert it into a symbol, so that we can animate the fish on the independent timeline and create a loop.
    1. Place the fish image on right side of the desktop screen.
    2. Activate the pin and animate the fish to the left side of the screen.
    3. Use the same duration to create another animation shortly afterwards to move the fish back to the start position.
    4. Select both animation tracks and add the easing “Ease Out: Bounce”
    5. Now to turn the fish around, first create a keyframe for the Scale(x) property (top property in the “Transform” section in the property inspector)
    6. Unlink the two values by clicking on the chain symbol on the left
    7. Move the playhead to the beginning of the second animation track and change orientation by setting Scale(x) to -100%.
    8. Go to the end of the the animation track and enter a trigger to play again from the beginning: sym.play(0);
  3. Create a visual parallax:
    1. Use the pin tool and set again the same duration as for moving the “scenes” element
    2. now select the laptop and mobile device image and move them a few inches to the left.
    3. Select the cloud and move it into the opposite direction. This way the assets closer to the front will move further in the same duration, the cloud in the back less far. You might have to test and tweak the distances a little to achieve the intended effect. Remember to deactivate the pin afterwards.

Adding interactivity and the Edge Commons Spotlight Overlay

  1. Start by placing the assets for the third scene next to the character onto the stage.
  2. Stop the character walking sequence using playback commands, this time selecting “Stop At” -> “talk”, so that he just faces the front.
  3. In order to be able to access the spotlight feature, you need to include jQuery and the Edge Commons library into your project. Therefore, copy the libs folder from the assets folder into your project folder.
  4. Use the scripts panel in the library to include both script files from disk, starting with jQuery. Therefore click on the “+” icon and search for the appropriate file (jquery-2.0.3.min.js). Then add the EdgeCommons.js file in the same way.
  5. The libs folder also includes a CSS file that is necessary for the right display of the overlay. Apparently you cannot use the scripts panel to include style sheets, so you have to do it manually:
    1. Open actions for stage (curly braces next to the stage name) and select compositionReady
    2. Here you enter the following code to use yepnope to load the style sheet:
      yepnope({load: “libs/spotlight.css”});
  6. Make sure the images for the spotlight lie in the /images folder of your project, otherwise copy them to this location.
  7. Now open actions for the first interactive element (S3_ani.png)
    1. select “click”
    2. and enter the following code:

          
// Spotlight Overlay with Image
// Configuration for spotlight overlay
var config = {
    width: 800,
    height: 600,
    type: "image",
    source: "images/screenshot_an.jpg"
};		

// Open the Edge Commons Spotlight Overlay with configuration
EC.Spotlight.open( config );

This will create a ‘config’ object, that contains all the relevant properties for the overlay. This way the overlay remains editable for different occasions. In the last line, the open function of the Edge Commons Spotlight module gets called with the config object as parameter.

Last but not least...

  1. You should remember to change the mouse cursor to pointer for the element, so that a user realises that he can interact.
  2. Repeat the last steps from open actions on for the other interactive element, choosing the equivalent other image source in the config object (source: "images/screenshot_rf.jpg").
  3. Now save the project and run it in the browser to see the result.

Congratulations, you have created an interactive scenery for story telling, featuring a sprite sheet animation and spotlight overlays.
