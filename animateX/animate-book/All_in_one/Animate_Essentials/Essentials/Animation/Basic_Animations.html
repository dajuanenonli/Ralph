Composing Animations

Basic Animations

First steps

This section will introduce you to the basics of creating animations in Edge Animate. You will learn how to work with keyframes and the timeline, master the pin tool and how to create and edit transitions. You will also learn how to produce more realistic animations with easings and motion paths.

Before we start...

  • You should again make yourself familiar with the UI, especially the Timeline
  • In case you find the terms still a bit confusing, you can use the included cheat sheet
  • Grab a cup of coffee

Recap: The Timeline

Animations in Edge Animate are represented on the Timeline. Here you can see all the Keyframes, Transitions and Animation Tracks for animated elements in a chronological order. Chronological means from left to right and everything that is placed underneath one another takes place at the same time. The red line underneath the playhead illustrates this, as it stretches across all tracks while indicating the current position of the playhead.

A very useful aspect of animating in Edge Animate is the instant preview in the design view. The display is related to the position of the playhead on the timeline, which means you can define the states visually on the stage and move the playhead back and forth to see the result. You can also play the composition to watch the animation already in the design view.

This gives you precise control over the choreography of your animations and allows you to adjust timing and duration as well as the arrangement of the single animation tracks. Another great advantage here is that you can edit and tweak the animations right on the timeline, which makes it even faster and easier to get the timing right.

One of the greatest features of Edge Animate is the fact that you can have more than one timeline. This is because each symbol has its own, independent timeline. All timelines work in the same way, but you can store different animations on each timeline, allowing you to create asynchronous animations and complex choreographies. We will come back to this a bit later in this chapter.

Keyframes

Animations in Edge Animate are realized through Keyframes. These keyframes are used to store a property value at a specific point in time on the timeline. Keyframes are represented by diamond shaped icons. When the playhead reaches the position of a keyframe, the corresponding property is set to the value stored in that particular keyframe. The change of values between two keyframes then creates an animation.

This leads to the question which values a keyframe could store. The answer is fairly easy: Almost every property in Animate can be stored in a keyframe and therefore be animated. Obvious examples are changing the position of an element or its size. But also background color, border radius or scaling can be animated. Whenever you see the keyframe diamond next to a property value, you can set a keyframe for this value. This gives quite the options for interesting animations and effects. Some of which will be shown later in examples.


Animating with keyframes

Now let's see keyframes in action and use them to create a simple animation. For starters, we want to move a plain element from left to right. The corresponding property is the position of the element on the X-axis, or in terms of CSS: left. We select the appropriate element that is to be animated, like in this case a red ellipse. We can then use the Keyframe icon next to the property to store that value as starting point by clicking on it.

Accordingly, a keyframe is created on the timeline at the current position of the playhead. In our example here it should be set at the very beginning of the timeline. Tip: You can always reposition keyframes on the timeline by simply dragging them to a new location. So if you accidentally set the keyframe someplace else, you can simply move it to the beginning of the timeline.

This is the starting point of our animation. As a next step, define the end position of the animation and store it in another keyframe. In order to do this, move the playhead on the timeline to the favored point in time, e.g. at 1,000 milliseconds. When we now move the element to its final position, a new keyframe is created automatically (as the Auto-Keyframe Mode is activated when setting the first keyframe). As you can see, this second keyframe stores the value of the new left position, and a Transition is created in between.

This Transition calculates each position between the two keyframes, so that the movement of the object appears to be steady and smooth. You can check the created animation right in the design view by using the play button or by hitting the space bar.

Note that you can use more then one keyframe for each animation track and that you can animate multiple properties at the same time. This would be the case when you change the position on the Y-axis as well, for instance. Go ahead and experiment a little with the different options to get a feeling for how these animations work.

Editing existing keyframes

Now that we have learned how to set keyframes, it is likely that you would also want to know how to change their values and how to remove them as well. We have already learned that we can simply move and re-position them on the timeline. This allows you for example to modify the duration of the animation: The closer two keyframes are to one another, the quicker the animation will be. Note that you can also copy and paste keyframes, for example to apply them to other elements as well.

When you want to change the value stored in a keyframe, you have to move the playhead to that location. Then you can either make the change in the design view - e.g. move the element around - or change the value immediately in the Timeline track by clicking on the value and typing in a new one.

Transitions

We have already encountered transitions in the previous section. Now let's talk a bit more about their functions and the options they offer us. Transitions are usually created automatically whenever we form a new animation, since in most of the cases, we want to generate the states in between two keyframes as a smooth animation.

There are some occasions, however, where you don't want to have such transitions but rather a rapid change from one state to another. Then you can switch off the Auto-Transition Mode in the Timeline Options (the green bar next to the stop watch icon). Now you only set the keyframes without adding a transition bar in between.

But what if you already have a transition bar? Well, that's not a problem at all. You can always add and remove transitions between two keyframes by right clicking on one of the two keyframes or the transition bar and choosing the option in the appearing menu.

Tip: With the last option in this menu, you can also invert the Transition. As a result, the two corresponding keyframes are simply switched, so that the animation now is going in the exact opposite direction as before. This can be very handy, since you are likely to accidentally animate in the wrong direction, especially when working with the Pin Tool.

Pin tool

The Pin Tool is a very useful feature for creating animations in Edge Animate. It might be a bit tricky to get used to at first, but once you've got the hang of it, you most likely don't want to animate without it anymore. We have already learned about activating and deactivating the Pin in the User Interface section. As a reminder: The most convenient way is by double-clicking on the playhead or using the "P" key on your keyboard.

The advantage of using the Pin is that it freezes all properties at its very location. You can then move the playhead to a different point in time to make alterations. All changes then appear as animation tracks between pin and playhead. Notice the "rubber band" that stretches between the pin and the playhead. It is indicating both the direction and duration of the animation. Yes, you can use the pin to animate into both directions: Towards the state fixed with the pin or from that state to the new state (the state created at the current playhead position).

The more intuitive way (and recommended especially for beginners) is to use the pin to store the initial state and animate towards a new state. Therefore we activate the pin (1) and move the playhead to the right (2). In this case, the bar ("rubber band") between pin and pleahead is yellow, with arrows to the right side.

The changes you make are always referring to the current position of the playhead. Since we have the initial state fixed with the pin, we can now go ahead and create the final state of the animation. For this example move the shape on the design view to the bottom right corner:

As you can see, the animation tracks and keyframes are added automatically, for every property that is affected. This is way more efficient than having to set keyframes yourself. And since you don't have to worry about that, you have more time to focus on the creative part.

When you now run the animation (short cut: space bar), you will see that the shape moves from the left hand side to the bottom right.

Let's have a quick look at the second option now as well. Therefore we start all over from the beginning with just a shape on the stage. Again, we activate the Pin, but this time we don't move the Playhead to the right but the Pin. Just like the Playhead, it can be moved by clicking and dragging it to a new position. Notice that the bar between Playhead and Pin now is in a blue color and the arrows are facing to the left.

Now we again move the shape to a new location in the bottom right of the stage. As before, animation tracks and keyframes are added automatically. This time, however, the animation will go into the opposite direction, as you can see in the preview.

This is because the Pin now stores the final state of the animation, while the initial state is created at the Playhead position. This can be rather confusing. Therefore we recommend using the former option only, so make sure that the "rubber band" is always yellow when using the Pin.

Reusing the duration of an animation

We have already learned that you can move the Playhead and the Pin individually to set the duration of your animation. In some cases you might want to use the exact same duration at a different point in time again. Here's another helpful tip for you: You can move the entire duration by clicking and dragging the "rubber band". This way, the Playhead and Pin will be moved simultaneously, while the distance between them remains the same.

Easings

We have now learned how to create basic animations in Edge Animate. But as you might have noticed, they appear to be rather unrealistic as they use the same speed throughout the entire movement. This kind of animation is called linear. In the "real world", such linear movements are very rare. Most of the times, movements accelerate and/or slow down. Fortunately, you can adopt such a behavior with the help of Easings.

In order to apply an easing function to an animation, you first have to select the according animation track. Then open up the Easings menu in the Timeline options section:

There are several options to choose from in order to get the right easing function for your animation. As we have already seen, the default setting is linear. Apart from that, we have the general options Ease In, Ease Out, Ease In Out and Swing. Once you select one of these options, you can set the precise curve, which means the way the easing behaves (not available for linear and swing).

Here are some examples:

It is quite easy to find the right options thanks to the curve drawn in the preview window. In general, Ease-In means that the animation becomes throughout the animation. This would be the right option for animating something that is falling down, for example. Ease-Out on the other hand results in a slowing down of the animation. A typical example from real life could be an elevator reaching the destination. As you have probably already guessed, Ease-In-Out is a combination of both: Starting slow, gaining speed and then slowing down in the end. The last option, Swing, is basically another Ease-Out.

Caution: Once you set Easings for an animation track, this easing becomes the new default for all further animation tracks! Be careful to set it back to linear to avoid unexpected animation behavior. Also note that the visual effect of the easing functions depend on the duration of the animation as well as the difference between the keyframe values.

Motion paths

Another option to create even more realistic animations is by using Motion Paths. The animations wie have covered so far referred to straight transitions from A to B. With Motion Paths, you can define the path along which the animation is to take place. Note that Motion Paths are only available for animating the Position of an element.

In order to create a motion path, select an element that is to be animated and change the Transition Mode in the Property Inspector to Motion Paths.

Once the Motion Path option is selected, you have only one Keyframe for the Location of the element instead of one for the X and one for the Y position. In order to create an animation, you can set Keyframes manually or use the pin tool. The path that is then being created will stretch from one location to the other in a straight line. This line is indicating the path along which the animation takes place.

You can customize this animation path by adding Anchors, that allow you to tweak the path and add curves. Click on the path to add a new Anchor and then drag this anchor to shape the path in order to create a curve.

You can then use the handles attached to the Anchor to determine the shape of the curve (so called "Bézier Curves"). Both distance and angle influence the shape of the curve. The good thing is that you see the result right away, so you can tweak around until you've found the right position.

Auto Orient

One last thing to mention regarding Motion Paths is the option to use Auto Orient for the animated element. You can find the corresponding option in the property inspector. When enabled, the orientation of the element changes along with the Motion Path, following its direction. Otherwise it simply moves along the path without changing the orientation.
