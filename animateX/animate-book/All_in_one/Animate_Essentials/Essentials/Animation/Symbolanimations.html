Composing Animations

Animations and Symbols

Getting into it

In this short part, you will get to know some of the more advanced aspects of creating animations with Edge Animate. We will discuss the potential of symbols with their independent timeline and talk about the concept of asynchronous animations.

Animations within a symbol

Especially when it comes to creating complex animation choreographies, the potentials of Symbols within Edge Animate becomes obvious. They allow us to stage asynchronous animations on their independent timeline. Symbols have their own scope, as we have already learned in the previous chapter. Still they can be linked to incidents on the main timeline, so that different animations can relate to each other.
That's worth having a closer look at.

Symbol Timeline

As we already know, symbols serve as a sort of composition within the composition. As such, they have their own Timeline, which allows us to create animations that are encapsulated within that particular symbol. This makes it possible to treat this animation as a closed entity, like a module, and even reuse or export it.

The Timeline itself works in the same way as with the main composition. You can create Keyframes and produce Transitions and so on, just like we just learned. In short: You can use the Timeline to create an independent animation choreography. One of the key aspects here is the option to enable or disable autoplay for the Symbol Timeline. This way you can decide whether the animation should start playing right away (as soon as it is placed on the stage) or whether it should wait for a certain event until starting to play. This is what you would call an asynchronous animation.

Now why would you need this in the first place? A good example is an animation loop, that can be contained within a Symbol. This way, the looped animation is independent of the main Timeline, which can therefore be used to create the main composition. We will show you how you can create such an animation loop shortly.

Another good example for the use of asynchronous animations is a button with different states. These different states can be animated on the symbol level and then be played on specific user events, like mouse-enter. You wouldn't want the button to change the states on its own now, would you? Keep in mind that you can also contain symbols within other symbols and therefore create complex constructions. Imagine an animated message box, which is going to appear on a certain event. This message box might then include some buttons and other symbols with further animations...and so on. You probably get the point.
