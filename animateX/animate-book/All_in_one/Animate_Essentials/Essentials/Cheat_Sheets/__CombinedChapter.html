Cheat Sheet Actions

The most important Actions

in Edge Animate

This cheat sheet will provide you with the most commonly used actions and code snippets for the use in Edge Animate. With each snippet, you will find a short description of its function and what you need to consider. Have fun!

Playback Actions

Controll the playback of your animations with these actions: 

          
// Play from current playhead position
sym.play();
       
// Play from label
// Indicate target label as string, e.g. "myLabel"
sym.play("myLabel");
        
// Play from point in time
// Indicate point in time in milliseconds, e.g. 1000
sym.play(1000);
        
// Play in opposite direction
// (also works with labels and timestamps)
sym.playReverse();
        
// Stop playback
sym.stop();
        
// Stop playback and move playhead to label
sym.stop("myLabel");
      
// Stop playback and move playhead to point in time
sym.stop(1000);
      
// Toggle play or pause the current timeline 
if (sym.isPlaying()) {
	sym.stop();
} else {
	sym.play();
}

Addressing symbols and elements

Using the Animate API to get the reference to the stage, symbols and elements: 

          
// Get reference to the stage (independent of current scope)
sym.getComposition().getStage();
        
// Get reference to a certain symbol (here: mySymbol) within the current scope, e.g. on the stage
sym.getSymbol("mySymbol");
      
// Get reference to a certain element (here: myElement) within the current scope
sym.$("myElement");
      
// Get reference to a symbol's containing element from within the symbol
sym.getSymbolElement("mySymbol");

Creating and deleting symbols

Using scripts to create and delete symbols at runtime 

          
// Creating a symbol instance inside a target element
// First parameter defines the symbol, second the target
sym.createChildSymbol("mySymbol", "myElement");
        
// Deleting a symbol instance
sym.deleteSymbol("mySymbol");
      
// Removing the content of a specific element
// e.g. removing a symbol instance as created above
sym.$("myElement").empty();

Variables and scopes

Using variables throughout different scopes 

          
// Setting a variable (on current scope)
// First parameter defines variable name, second its value
sym.setVariable("myVariable", value);
        
// Retrieving a variable
sym.getVariable("myVariable");
      
// EXAMPLE:
// Creating variable "count" on stage level, 
// then passing it into the symbol "mySymbol"
      
// (on stage level)
var count = 0;
      
sym.getSymbol("mySymbol").setVariable("count", count);

// (within the symbol) 
var count = sym.getVariable("count");

Adding content

Using scripts to add or alter content at runtime 

          
// Changing the html inside an element 
// e.g. to change text inside a text element
sym.$("myElement").html("Your new text goes here");

// Adding content to existing elements
// e.g. adding further text
sym.$("myElement").append("Additional text goes here");

Links

Navigate to a different location with these actions: 

          
// Navigate to a new URL in the current window
window.open("http://www.adobe.com", "_self");
        
// Navigate to a new URL in a new window
window.open("http://www.adobe.com", "_blank");
      
// Email to certain address
window.location.href("mailto:destination@address.com");

Hide/Show

Hide or reveal hidden elements and symbols with these actions: 

          
// Hide an element 
sym.$("Element_Name").hide();
        
// Show an element 
sym.$("Element_Name").show();
      
// Set a toggle to hide or show an element 
if (sym.$("Element_Name").is(":visible")) {
	sym.$("Element_Name").hide();
} else {
	sym.$("Element_Name").show();
}
Cheat Sheet Events

The most important Events

in Edge Animate

This cheat sheet will provide you with the Events you will most likely encounter using Edge Animate. With each Event, you will find a short description of its purpose and what you need to consider. Enjoy!

Stage Events

  • creationComplete: This event fires when the stage instance is created. Similar to compositionReady, this event can be used to set and define initialization statements.
  • beforeDeletion: This event can be used to clean up before the stage is being removed.
  • compositionReady: The compositionReady event fires after the composition is initialized. This allows you to further prepare the composition with code, e.g. by setting variables or loading extensions like external CSS.
  • scroll: This event fires when the user scrolls within the composition.
  • keydown/keyup: This event fires when a key is pressed or released, respectively.
  • orientationchange: This event is specific for mobile devices. It fires when the device is rotated, e.g. from portrait to landscape view.
  • resize: This event fires as soon as the size of the browser window changes.
  • onError: This event is a global error handler and is always called when the code for some other event produces a JavaScript error.

Element Events

  • click: This event is fired when the user clicks on the element.
  • dblclick: This event fires when the user uses a double click on the element.
  • mouseover: This event fires when the mouse cursor enters the area of the element.
  • mouseout: This event fires when the mouse cursor leaves the area of the element.
  • mousedown: This event fires when the user uses a double click on the element.
  • mouseup: This event fires when the user uses a double click on the element.
  • mousemove: This event runs continously while the mouse moves over the element.
  • touchstart: This event fires when a finger touches the element on a touch screen.
  • touchmove: This event fires continously when the finger moves over the touch screen after touchstart.
  • touchend: This event fires when the user releases the finger from touch screen.
  • mouseenter/mouseleave: These events are similar to mouseover/mouseout, but in these cases the event is only fired once within the boundaries of the element.
  • focus: This event is fired when the element gets the focus.
  • swipeleft/right: This event fires when the user finger-swipes the element on a touch screen.

Symbol Events

  • creationComplete: This event fires when a symbol instance is being created on the stage. This allows you to define initializing statements.
  • beforeDeletion: This event can be used to clean up before the symbol is being removed from the stage.

Timeline Events

  • update: This event is fired continously when the playhead moves on the timeline, i.e. the timeline is playing.
  • play: This event is triggered when the playhead starts to move.
  • complete: This event fires when the playhead reaches the end of the timeline.
  • stop: This event fires when the playhead is being stopped.

Cheat Sheet Terminology

The most important terms and expressions

for Edge Animate

This cheat sheet will provide you with the most important terms and expressions used in regards to Edge Animate. This will help you get an easier access to the tool, its mechanisms and this guide book. Enjoy!

Runtime

Refers to the composition being executed in a browser as opposed to the preview in Edge Animate during -> design time.

Design time

Representation of the Animate composition in Edge Animate.

Trigger

A trigger allows you to define a point on the Timeline on which an action is then being executed.

Label

Marker on the Timeline that can be used as a reference, e.g. within playback commands or triggers.

Action

Actions are JavaScript code snippets that effect for example the playback of the Timeline or define the result of a click event.

Scope

The scope refers to the current context, for example being on the stage level or within a symbol.

Stage

The stage is your design canvas and the top most element of your composition. The stage has unique events and settings.

Stage Level

The stage level refers in general to the top most scope of your Animate composition.

Symbol

Symbols are powerful entities in Edge Animate. They encapsulate elements and actions and have an independent timeline. They can be regarded as some kind of sub-composition (similar to movie-clips in Flash).

Transition

A transition is the computed animation sequence between two keyframes, so that each step is interpolated.

Keyframe

A keyframe sets the starting or ending point of an animation by storing the value of a property on the timeline.
